O que é Middleware?

Middleware é uma função que tem acesso ao objeto de requisição (req), objeto de resposta (res) e à função next() no ciclo de requisição-resposta de uma aplicação Express.js.

Ele pode modificar a requisição, resposta ou encerrar o ciclo sem passar para a próxima função.

Por que usar Middleware?

Os middlewares permitem a manipulação de requisições antes que cheguem ao controlador final.

Principais usos do Middleware:

    Processamento de requisições antes de chegar ao roteador.

    Autenticação e autorização de usuários.

    Manipulação de erros centralizada.

    Registro (logging) de requisições.

    Compressão e otimização de respostas.

Tipos de Middleware

No Express.js, existem diferentes tipos de middleware:

Middleware de Aplicação

Executado para todas as requisições na aplicação.

Middleware de Rota

Executado apenas para rotas específicas.

Middleware Incorporado

Middlewares que já vêm com o Express, como express.json() e express.urlencoded().

Middleware de Terceiros

Middlewares criados por terceiros, como Morgan (para logs) e CORS (para requisições entre domínios).

Middleware de Manipulação de Erros

Utilizado para capturar e tratar erros globalmente.

Como criar um Middleware?

Um middleware em Express.js é uma função que recebe três parâmetros: req, res e next. Ele pode modificar a requisição, enviar uma resposta ou chamar next() para passar para o próximo middleware.

Exemplo de Middleware:


function meuMiddleware(req, res, next) {
    console.log("Middleware ativado!");
    next(); // Passa para o próximo middleware ou rota
}

Middleware no Express.js

Para usar um middleware no Express, basta adicioná-lo ao app com app.use() ou diretamente em rotas específicas.

O que é Multer?

Multer é um middleware para Express.js que permite o upload de arquivos. Ele processa multipart/form-data, que é o formato necessário para enviar arquivos via formulários HTML.

Por que usar o Multer?

O Multer facilita o gerenciamento de arquivos enviados pelos usuários. Sem ele, o Express não consegue lidar com uploads de forma eficiente.

Vantagens do Multer:

    Processa arquivos rapidamente no servidor.p

    Permite armazenar arquivos no disco ou na memória.

    Suporta múltiplos arquivos simultaneamente.

Como instalar o Multer?

Para instalar o Multer em um projeto Node.js, basta rodar:

npm install multer

Isso adiciona o Multer às dependências do projeto.

Como configurar o Multer?

O Multer permite armazenar arquivos de duas formas:

1 Armazenamento no Disco:

Os arquivos são salvos no servidor.

2 Armazenamento na Memória:

Os arquivos são armazenados temporariamente.

Outras configurações do Multer:

    Pasta de destino: Define onde os arquivos serão salvos.

    Nome do arquivo: Pode ser alterado automaticamente.

    Filtros: Permitem restringir tipos de arquivos aceitos (exemplo: apenas imagens).

Como fazer upload com Multer?

O Multer pode ser usado de diferentes maneiras para lidar com uploads:

Upload de um único arquivo:

Permite enviar apenas um arquivo por vez.

Upload de múltiplos arquivos:

Aceita vários arquivos ao mesmo tempo.

Upload de arquivos com validação:

Pode verificar tamanho, tipo e extensão antes de aceitar o upload.
